# Plan 11 — PWA Shell + Offline Support

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn WhatToEat 2.0 into a fully installable Progressive Web App. This means: a correctly-formed web app manifest wired to Next.js metadata, app icons (192 px, 512 px, maskable) generated from a single source SVG, a Workbox-powered service worker with per-route caching strategies, an offline fallback page that uses design-system tokens, iOS Safari "Add to Home Screen" polish (viewport-fit, safe-area-inset, standalone display tweaks), a "new version available" toast via Sonner that fires exactly once per SW activation, CI guards that assert the manifest validates and the SW is emitted, and Playwright + unit tests that cover the critical behaviors.

**Architecture:** The service worker is registered by a small client component (`src/components/pwa/SwRegister.tsx`) that mounts once in the root layout. Workbox runs fully at build time via `workbox-build` (called from a custom Next.js plugin in `next.config.ts`) — this avoids the `@ducanh2912/next-pwa` abstraction layer, which as of 2026 has not been updated for Next.js 15 App Router's revised `_next/static` manifest layout. The hand-rolled approach gives precise control over which routes go through which strategy, and the code surface is small enough to maintain without a wrapper library. The offline page lives at `src/app/offline/page.tsx` and is pre-cached at SW install time so it is always available on disk regardless of network status.

**Implements after T1 + T5 land.** T1 is required because the offline page uses token-derived Tailwind utilities and the Sonner `<Toaster>` already mounted in the root layout. T5 is required because the pantry route is the one offline-readable route (the SW caches its HTML shell and its SWR data) — the caching strategy for `/pantry` must treat the page shell as StaleWhileRevalidate rather than NetworkFirst, and the SW must pre-cache the page's JS chunks.

**Why hand-rolled Workbox, not @ducanh2912/next-pwa?** Three reasons. First, `@ducanh2912/next-pwa` wraps an older Workbox build pipeline that does not correctly map the Next.js 15 App Router build manifest (`.next/app-build-manifest.json`) to precache entries; the resulting service worker silently omits App Router JS chunks. Second, its "disable in dev" logic relies on `process.env.NODE_ENV` injection at require-time in a way that conflicts with Next.js 15's Turbopack fast-refresh path. Third, the project already owns its `next.config.ts` plugin slot (used by Plan 12 for Sentry source maps), making it straightforward to add a second plugin that runs `workbox-build` after `next build` without an extra abstraction. The downside is ~120 lines of glue code that must be maintained when Workbox releases a major version — acceptable for a project of this scale.

**Tech Stack:** Bun, Next.js 15 (App Router), React 19, Tailwind CSS v3 (tokens from T1), `workbox-build` (Google), `workbox-window` (client-side SW registration helper), `sonner` (already in T1), Vitest, Playwright (or `lighthouse-ci` for the PWA audit), `sharp` (icon generation from SVG).

**Spec reference:** `docs/superpowers/specs/kanban-tasks.yaml` — T11 entry (installable PWA on iOS Safari + Chromium, pantry view readable offline, graceful empty state for offline recommendation attempts).

```ts
#!/usr/bin/env bun
/**
 * scripts/generate-icons.ts
 *
 * Generates PWA icon PNGs from a single source SVG using sharp.
 * Produces four output files:
 *   public/icons/icon-192.png          — standard icon, 192x192, transparent bg
 *   public/icons/icon-512.png          — standard icon, 512x512, transparent bg
 *   public/icons/icon-maskable-192.png — maskable icon, 192x192, amber bg + 10% padding
 *   public/icons/icon-maskable-512.png — maskable icon, 512x512, amber bg + 10% padding
 *
 * Run: bun run scripts/generate-icons.ts
 * Prerequisite: bun add -d sharp
 */

import sharp from 'sharp';
import path from 'node:path';
import fs from 'node:fs';

const ROOT = path.resolve(import.meta.dir, '..');
const ICONS_DIR = path.join(ROOT, 'public', 'icons');
const SOURCE_SVG = path.join(ICONS_DIR, 'icon.svg');

// Brand amber (#D97706) as RGB components
const AMBER_R = 217;
const AMBER_G = 119;
const AMBER_B = 6;

// The maskable safe zone is 10% inset on each side (W3C recommendation).
// We add amber background + shrink the inner graphic to fit within the safe area.
const MASKABLE_PADDING_RATIO = 0.1;

async function generateStandard(size: number): Promise<void> {
  const out = path.join(ICONS_DIR, `icon-${size}.png`);
  await sharp(SOURCE_SVG)
    .resize(size, size, { fit: 'contain', background: { r: 0, g: 0, b: 0, alpha: 0 } })
    .png()
    .toFile(out);
  console.log(`  Generated ${path.relative(ROOT, out)}`);
}

async function generateMaskable(size: number): Promise<void> {
  const out = path.join(ICONS_DIR, `icon-maskable-${size}.png`);

  // Inner graphic fits within the safe zone
  const padding = Math.round(size * MASKABLE_PADDING_RATIO);
  const innerSize = size - padding * 2;

  // 1. Render the SVG into a PNG at innerSize x innerSize with transparent bg
  const innerPng = await sharp(SOURCE_SVG)
    .resize(innerSize, innerSize, { fit: 'contain', background: { r: 0, g: 0, b: 0, alpha: 0 } })
    .png()
    .toBuffer();

  // 2. Compose: amber square canvas + centered inner PNG
  await sharp({
    create: {
      width: size,
      height: size,
      channels: 4,
      background: { r: AMBER_R, g: AMBER_G, b: AMBER_B, alpha: 1 },
    },
  })
    .composite([{ input: innerPng, gravity: 'center' }])
    .png()
    .toFile(out);

  console.log(`  Generated ${path.relative(ROOT, out)}`);
}

async function main(): Promise<void> {
  if (!fs.existsSync(SOURCE_SVG)) {
    console.error(`Source SVG not found: ${SOURCE_SVG}`);
    process.exit(1);
  }

  fs.mkdirSync(ICONS_DIR, { recursive: true });

  console.log('Generating PWA icons...');
  await generateStandard(192);
  await generateStandard(512);
  await generateMaskable(192);
  await generateMaskable(512);
  console.log('Done.');
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

### Task 3: Web app manifest at `/public/manifest.webmanifest`

**Files:** `public/manifest.webmanifest`

The manifest is a static JSON file served from the `/public` directory (Next.js copies it verbatim to `/_next/public/` and serves it at the root URL). It uses `.webmanifest` extension so browsers receive the correct `application/manifest+json` MIME type, which is required by the Lighthouse "installable" check.

Key decisions baked into this manifest:

- `display: "standalone"` — the app launches without browser chrome, and the status bar appears over the app canvas (important for the safe-area CSS in Task 8).
- `background_color` and `theme_color` are both set to the brand amber hex `#D97706`, matching the `--accent` token from T1. This color fills the splash screen on Android and iOS during launch.
- `start_url: "/?source=pwa"` — the `source` query parameter lets analytics distinguish installs from direct web visits without affecting routing.
- `scope: "/"` — the SW controls the entire origin.
- Four icons are listed: 192 standard, 192 maskable, 512 standard, 512 maskable. Chromium uses the 512 for high-DPI screens and the splash screen; iOS uses 192 for the home screen icon.
- `prefer_related_applications: false` — tells Chrome not to show a banner for a native app instead of the web app.
- `categories: ["food", "health", "productivity"]` — optional but helps app stores and browser UI classify the install.

### Task 4: Workbox build plugin wired into `next.config.ts`

**Files:** `next.config.ts`

This is the most architecturally important task. The plugin runs a `workbox-build` `injectManifest` call after `next build` completes, reading the Next.js app build manifest to compute the set of static assets to precache. The service worker **source** lives at `src/app/sw.ts` (TypeScript, compiled separately in Task 5); the plugin outputs the fully-formed service worker to `public/sw.js`.

`injectManifest` mode is chosen over `generateSW` because it lets us write custom service worker logic (NetworkOnly for API routes, server-action POST guard, SW versioning, skipWaiting message channel) rather than relying entirely on Workbox's opinionated defaults.

The plugin is disabled when `process.env.NODE_ENV !== 'production'` or when `process.env.NEXT_PWA_DISABLED === '1'`. In development, the service worker is not registered at all (see Task 7). This avoids the classic "stale SW breaks hot reload" developer experience problem.

- [ ] **Step 1: Write the Workbox plugin function**

Create `scripts/workbox-plugin.ts`:

```ts
/**
 * scripts/workbox-plugin.ts
 *
 * A minimal Next.js config plugin that runs workbox-build injectManifest
 * after the Next.js build completes. Called from next.config.ts.
 *
 * Inputs:
 *   src/app/sw.ts  (TypeScript service worker source — must be pre-compiled)
 *   .next/app-build-manifest.json  (Next.js 15 App Router build manifest)
 *
 * Output:
 *   public/sw.js  (precache-injected service worker, ready to register)
 *
 * The plugin is a no-op in dev (NODE_ENV !== 'production') or when
 * NEXT_PWA_DISABLED=1 is set.
 */

import type { NextConfig } from 'next';

export function withPWA(nextConfig: NextConfig): NextConfig {
  return {
    ...nextConfig,
    webpack(webpackConfig, context) {
      // Pass through any existing webpack customization from the base config.
      const upstream =
        typeof nextConfig.webpack === 'function'
          ? nextConfig.webpack(webpackConfig, context)
          : webpackConfig;

      // Only wire up Workbox on the server-side build pass (not the client pass)
      // and only in production. workbox-build is a heavy Node.js operation that
      // must run after all assets are emitted, so we hook the 'afterEmit' compiler
      // hook rather than adding a loader.
      if (!context.isServer || context.dev || process.env.NEXT_PWA_DISABLED === '1') {
        return upstream;
      }

      // The afterEmit hook fires after all assets from this compilation have
      // been written to disk. At this point .next/app-build-manifest.json and
      // all static JS chunks exist on disk.
      upstream.hooks.afterEmit.tapPromise('WorkboxBuildPlugin', async () => {
        const { injectManifest } = await import('workbox-build');
        const path = await import('node:path');
        const esbuild = await import('esbuild');
        const fs = await import('node:fs');

        const root = process.cwd();
        const swSrc = path.join(root, 'src', 'app', 'sw.ts');
        const swCompiledJs = path.join(root, '.next', 'sw-source.js');
        const swOut = path.join(root, 'public', 'sw.js');

        // Step A: Compile sw.ts to a single JS file using esbuild.
        // We cannot use tsc here because workbox-build needs a plain JS file
        // as its swSrc input. esbuild is bundled with Next.js so no extra dep.
        await esbuild.build({
          entryPoints: [swSrc],
          bundle: true,
          format: 'iife',
          target: 'chrome90',
          platform: 'browser',
          outfile: swCompiledJs,
          // Do not minify — the SW is large and we want readable error stacks.
          minify: false,
          // Workbox libraries are CDN-imported inside sw.ts using importScripts;
          // they must NOT be bundled here.
          external: [
            'workbox-core',
            'workbox-routing',
            'workbox-strategies',
            'workbox-precaching',
            'workbox-expiration',
          ],
          define: {
            'process.env.NODE_ENV': '"production"',
            __SW_VERSION__: JSON.stringify(Date.now().toString()),
          },
        });

        // Step B: Read .next/app-build-manifest.json to get all App Router
        // page JS chunk paths (Next.js 15 format).
        const appBuildManifestPath = path.join(root, '.next', 'app-build-manifest.json');
        const staticManifestPath = path.join(root, '.next', 'build-manifest.json');

        const appManifest = JSON.parse(
          fs.readFileSync(appBuildManifestPath, 'utf8'),
        ) as Record<string, string[]>;

        const buildManifest = JSON.parse(
          fs.readFileSync(staticManifestPath, 'utf8'),
        ) as { pages: Record<string, string[]>; devFiles: string[]; ampDevFiles: string[]; polyfillFiles: string[]; lowPriorityFiles: string[] };

        // Collect all unique /_next/static/** paths from both manifests.
        const staticChunks = new Set<string>();

        // App Router chunks
        for (const chunks of Object.values(appManifest)) {
          for (const chunk of chunks) {
            staticChunks.add(chunk.startsWith('/') ? chunk : `/_next/${chunk}`);
          }
        }

        // Pages Router chunks (if any coexist; typically empty in full App Router projects)
        for (const chunks of Object.values(buildManifest.pages ?? {})) {
          for (const chunk of chunks) {
            staticChunks.add(chunk.startsWith('/') ? chunk : `/_next/${chunk}`);
          }
        }

        // Step C: Run workbox-build injectManifest.
        // This takes swCompiledJs, finds the self.__WB_MANIFEST placeholder,
        // replaces it with the computed precache manifest, and writes public/sw.js.
        const { count, size } = await injectManifest({
          swSrc: swCompiledJs,
          swDest: swOut,
          globDirectory: path.join(root, 'public'),
          // We manually pass additionalManifestEntries for /_next/static assets
          // because workbox-build's glob cannot reach into .next/
          globPatterns: [
            // Static files from /public (favicon, icons, manifest)
            '**/*.{png,svg,ico,webmanifest,woff2,woff}',
          ],
          additionalManifestEntries: Array.from(staticChunks).map((url) => ({
            url,
            revision: null, // Next.js already includes content hash in filenames
          })),
          // Do NOT precache the offline page via glob — we add it explicitly in sw.ts
          // via precacheAndRoute([{ url: '/offline', revision: '1' }]).
          // Do NOT precache API routes or server actions.
          globIgnores: [
            // Exclude the SW itself from its own precache manifest
            'sw.js',
            'sw.js.map',
          ],
          // Maximum size of a single asset to precache. Assets larger than this
          // are fetched on-demand only. 4 MB is generous for a web app.
          maximumFileSizeToCacheInBytes: 4 * 1024 * 1024,
        });

        console.log(
          `[WorkboxPlugin] Injected precache manifest: ${count} entries, ~${(size / 1024).toFixed(1)} KB`,
        );
      });

      return upstream;
    },
  };
}
```

import { withPWA } from './scripts/workbox-plugin';

const baseConfig: NextConfig = {
  // Enforce that server components do not accidentally import client-only modules.
  // This is on by default in Next.js 15; listed here for clarity.
  serverExternalPackages: [],

  // Disable the X-Powered-By header (minor security hardening).

  async headers() {
    return [
      {
        // Ensure the browser reads the SW scope correctly.
        // Service-Worker-Allowed: / tells the browser the SW may control the root.
        source: '/sw.js',
        headers: [

### Task 5: Service worker source — `src/app/sw.ts`

**Files:** `src/app/sw.ts`

This is the service worker itself. It is a TypeScript file that uses Workbox CDN imports via `importScripts`. The build plugin (Task 4) compiles it with esbuild, then `workbox-build injectManifest` replaces the `self.__WB_MANIFEST` placeholder with the computed list of static assets to precache.

**Caching strategy rationale:**

| Route pattern | Strategy | Reasoning |
|---|---|---|
| `/offline` | Precache (SW install) | Always available offline by design |
| `/_next/static/**` | Precache (SW install) | Content-hashed filenames, immutable; fastest load from cache |
| `*.png, *.svg, *.ico, *.webmanifest` | Precache (SW install) | Small static assets, immutable |
| HTML navigation (not /api/*) | NetworkFirst (7 s timeout) | Fresh data preferred; cached HTML served on network failure |
| `/pantry` HTML specifically | StaleWhileRevalidate | Pantry must be readable offline; show cached HTML instantly |
| Images not in precache | StaleWhileRevalidate (30 days, 50 entries) | Good performance, tolerate stale images |
| `/api/**` | NetworkOnly | Never serve API responses from cache; data must be fresh |
| POST requests (server actions) | NetworkOnly | Mutations must never hit cache |

The SW also implements a `message` channel for the "new version available" toast:

1. When a new SW is waiting, `workbox-window` on the client fires `waiting`.
2. The client component sends `{ type: 'SKIP_WAITING' }` after the user clicks "Refresh" in the toast (or automatically, depending on UX choice — see Task 7).
3. The SW receives this message and calls `self.skipWaiting()`, then `clients.claim()` runs automatically because it is called at the top of the activate handler.

- [ ] **Step 1: Create `src/app/sw.ts`**

```ts
/**
 * src/app/sw.ts
 *
 * WhatToEat 2.0 — Service Worker (Workbox injectManifest pattern)
 *
 * This file is compiled by esbuild in scripts/workbox-plugin.ts, then
 * workbox-build injectManifest replaces self.__WB_MANIFEST with the
 * computed precache manifest. Output lands at public/sw.js.
 *
 * DO NOT import this file from any Next.js page or server component.
 * It runs in a ServiceWorkerGlobalScope, not in the browser document.
 *
 * Workbox is loaded via importScripts from the official CDN so that esbuild
 * does not bundle it (the external: [...] setting in workbox-plugin.ts).
 */

// @ts-ignore — importScripts is available in SW global scope
importScripts('https://storage.googleapis.com/workbox-cdn/releases/7.1.0/workbox-sw.js');

// TS cast so we get type hints from the global workbox object.
declare const workbox: typeof import('workbox-sw');
declare const self: ServiceWorkerGlobalScope & {
  __WB_MANIFEST: Array<{ url: string; revision: string | null }>;
  skipWaiting(): void;
};

// Fail loudly if Workbox failed to load (e.g. offline during SW install).
if (!workbox) {
  console.error('[SW] Workbox failed to load — PWA offline support will not function.');
}

// ---------------------------------------------------------------------------
// 1. Precache — static assets injected at build time
// ---------------------------------------------------------------------------
// self.__WB_MANIFEST is replaced by workbox-build injectManifest.
// It contains all /_next/static/** chunks and public/** assets computed
// from the Next.js build manifests.

workbox.precaching.precacheAndRoute(self.__WB_MANIFEST ?? []);

// Always precache the offline fallback page explicitly.
// revision: '1' means "always re-fetch when the SW updates".
// If you update the offline page UI, bump this revision string.
workbox.precaching.precacheAndRoute([
  { url: '/offline', revision: '2' },
]);

// ---------------------------------------------------------------------------
// 2. Skip waiting on message from client
// ---------------------------------------------------------------------------
// workbox-window sends { type: 'SKIP_WAITING' } when the user confirms
// the "new version available" prompt. We call skipWaiting here so the
// new SW takes control of all tabs immediately after the page reloads.

self.addEventListener('message', (event) => {
  if (event.data && event.data.type === 'SKIP_WAITING') {
    self.skipWaiting();
  }
});

// ---------------------------------------------------------------------------
// 3. Claim clients on activate
// ---------------------------------------------------------------------------
// After skipWaiting, the new SW activates but does not control existing tabs
// until they reload — unless we call clients.claim(). This ensures the SW
// starts intercepting fetches in the current tab immediately.

workbox.core.clientsClaim();

// ---------------------------------------------------------------------------
// 4. NetworkOnly for all POST requests (server actions + mutations)
// ---------------------------------------------------------------------------
// Next.js server actions use POST to /. Mutations must never be served
// from cache — a cached POST response could silently replay a stale action.

workbox.routing.registerRoute(
  ({ request }) => request.method === 'POST',
  new workbox.strategies.NetworkOnly(),
);

// ---------------------------------------------------------------------------
// 5. NetworkOnly for /api/** routes
// ---------------------------------------------------------------------------
// API responses are live data (recommendations, stream responses). Caching
// them would surface stale meal suggestions or phantom auth states.

workbox.routing.registerRoute(
  ({ url }) => url.pathname.startsWith('/api/'),
  new workbox.strategies.NetworkOnly(),
);

// ---------------------------------------------------------------------------
// 6. StaleWhileRevalidate for the /pantry HTML shell
// ---------------------------------------------------------------------------
// The pantry page is the one offline-readable surface. Users should be able
// to view their pantry even when the network is down (SWR data was cached
// by the application layer; the SW only needs to serve the HTML shell).
// StaleWhileRevalidate means: serve the cached shell instantly, then update
// the cache from the network in the background.
//
// We scope this to 'navigate' requests for /pantry only to avoid
// accidentally caching other navigation responses with stale-while-revalidate.

workbox.routing.registerRoute(
  ({ url, request }) =>
    request.mode === 'navigate' && url.pathname.startsWith('/pantry'),
  new workbox.strategies.StaleWhileRevalidate({
    cacheName: 'pantry-html-cache',
    plugins: [
      new workbox.expiration.ExpirationPlugin({
        maxEntries: 5,
        maxAgeSeconds: 7 * 24 * 60 * 60, // 1 week
      }),
    ],
  }),
);

// ---------------------------------------------------------------------------
// 7. NetworkFirst for all other HTML navigations
// ---------------------------------------------------------------------------
// All non-pantry navigations use NetworkFirst with a 7-second timeout.
// If the network fails (offline or timeout), serve the cached version of
// that URL if available, otherwise serve the /offline fallback page.

const networkFirstHandler = new workbox.strategies.NetworkFirst({
  cacheName: 'html-navigation-cache',
  networkTimeoutSeconds: 7,
  plugins: [
    new workbox.expiration.ExpirationPlugin({
      maxEntries: 30,
      maxAgeSeconds: 24 * 60 * 60, // 1 day
    }),
  ],
});

workbox.routing.registerRoute(
  ({ request }) => request.mode === 'navigate',
  async (context) => {
    try {
      return await networkFirstHandler.handle(context);
    } catch {
      // Network failed and no cached version — serve the offline fallback.
      const cache = await caches.open('workbox-precache-v2');
      const offlineFallback = await cache.match('/offline');
      if (offlineFallback) return offlineFallback;
      // Last resort: return a minimal inline HTML response.
      return new Response(
        '<html><body><p>You are offline. Please reconnect and try again.</p></body></html>',
        { headers: { 'Content-Type': 'text/html' } },
      );
    }
  },
);

// ---------------------------------------------------------------------------
// 8. StaleWhileRevalidate for /_next/static/** (already in precache, belt-and-suspenders)
// ---------------------------------------------------------------------------
// Static JS/CSS chunks are content-hashed by Next.js and thus immutable.
// If a chunk somehow misses the precache manifest (e.g. a dynamic import
// loaded for the first time), catch it here and cache it for the session.

workbox.routing.registerRoute(
  ({ url }) => url.pathname.startsWith('/_next/static/'),
  new workbox.strategies.StaleWhileRevalidate({
    cacheName: 'next-static-cache',
  }),
);

// ---------------------------------------------------------------------------
// 9. StaleWhileRevalidate for images not in precache
// ---------------------------------------------------------------------------

workbox.routing.registerRoute(
  ({ request }) => request.destination === 'image',
  new workbox.strategies.StaleWhileRevalidate({
    cacheName: 'image-cache',
    plugins: [
      new workbox.expiration.ExpirationPlugin({
        maxEntries: 50,
        maxAgeSeconds: 30 * 24 * 60 * 60, // 30 days
      }),
    ],
  }),
);
```

### Task 7: SW registration client component + "new version available" toast

**Files:** `src/components/pwa/SwRegister.tsx`, `src/components/pwa/index.ts`

The `SwRegister` component mounts once in the root layout (Task 8). It uses `workbox-window`'s `Workbox` class to register the service worker and listen for the `waiting` event, which fires when a new SW has installed but is blocked from activating because an older SW controls the page. A separate "offline indicator" badge is shown in the bottom-left corner when `navigator.onLine` is false. This uses a `useEffect` that listens to the `online` and `offline` window events. The badge uses the `--err` token color (red) to match the status palette from T1.

'use client';

import { useEffect, useRef, useState } from 'react';
import { toast } from 'sonner';

/**
 * SwRegister — mounts once in the root layout.
 *
 * Responsibilities:
 *   1. Register /sw.js via workbox-window in production.
 *   2. Listen for the 'waiting' lifecycle event (new SW installed, waiting to activate).
 *   3. Show a Sonner toast with a "Refresh" CTA when a new version is waiting.
 *   4. On "Refresh" click: post SKIP_WAITING to the waiting SW, then reload.
 *   5. Show a small offline indicator badge when navigator.onLine is false.
 *
 * Disabled in dev (process.env.NODE_ENV !== 'production') to avoid conflicts
 * with Next.js HMR. The SW file is not emitted in dev anyway.
 */

// We import workbox-window lazily so it is not included in the SSR bundle.
// The dynamic import resolves only on the client after mount.

type WorkboxInstance = import('workbox-window').Workbox;

const SW_PATH = '/sw.js';

// How long to wait before showing the update toast after detecting a waiting SW.
// A small delay prevents the toast from appearing before the page finishes loading.
const TOAST_DELAY_MS = 3000;

function useServiceWorker() {
  const wbRef = useRef<WorkboxInstance | null>(null);
  const [isOnline, setIsOnline] = useState(true);

  useEffect(() => {
    // SSR guard — this effect only runs in the browser.
    if (typeof window === 'undefined') return;

    // Offline/online status
    setIsOnline(navigator.onLine);
    const handleOnline = () => setIsOnline(true);
    const handleOffline = () => setIsOnline(false);
    window.addEventListener('online', handleOnline);
    window.addEventListener('offline', handleOffline);

    // Only register the service worker in production.
    if (
      process.env.NODE_ENV !== 'production' ||
      !('serviceWorker' in navigator)
    ) {
      return () => {
        window.removeEventListener('online', handleOnline);
        window.removeEventListener('offline', handleOffline);
      };
    }

    let didCleanup = false;

    (async () => {
      // Dynamic import keeps workbox-window out of the SSR bundle entirely.
      const { Workbox } = await import('workbox-window');
      if (didCleanup) return;

      const wb = new Workbox(SW_PATH);
      wbRef.current = wb;

      // 'waiting' fires when a new SW has installed but is blocked from activating.
      // This happens on every subsequent visit after the first install, or when
      // the user has another tab open with the old SW.
      wb.addEventListener('waiting', () => {
        const toastId = setTimeout(() => {
          toast('A new version of WhatToEat is ready.', {
            id: 'sw-update',
            duration: Infinity, // Keep visible until user acts.
            action: {
              label: 'Refresh',
              onClick: () => {
                // Tell the waiting SW to skip waiting and take control.
                wb.messageSkipWaiting();
                // Reload the page so the new SW activates for this tab.
                window.location.reload();
              },
            },
            onDismiss: () => {
              // User dismissed without refreshing — the old SW keeps control
              // until the next navigation or tab close. This is fine.
            },
          });
        }, TOAST_DELAY_MS);

        // Clean up the timer if the component unmounts before the toast fires.
        return () => clearTimeout(toastId);
      });

      // 'activated' fires the first time the SW takes control after a fresh install.
      // We do not show a toast here — the page is already on the latest version.
      wb.addEventListener('activated', (event) => {
        if (!event.isUpdate) {
          // First install — no toast needed.
          return;
        }
        // An updated SW just activated (user clicked Refresh or opened a new tab).
        // Dismiss any lingering update toast.
        toast.dismiss('sw-update');
      });

      // 'controlling' fires when the SW starts controlling this page.
      // Log for debugging; no UI action needed.
      wb.addEventListener('controlling', () => {
        console.debug('[SW] Now controlling this page.');
      });

      await wb.register();
    })();

    return () => {
      didCleanup = true;
      window.removeEventListener('online', handleOnline);
      window.removeEventListener('offline', handleOffline);
    };
  }, []);

  return { isOnline };
}

export function SwRegister() {
  const { isOnline } = useServiceWorker();

  if (isOnline) return null;

  // Offline indicator badge — always visible when the device is offline.
  // Positioned in the bottom-left corner above any navigation UI.
  return (
    <div
      role="status"
      aria-live="polite"
      aria-label="You are currently offline"
      className="fixed bottom-4 left-4 z-50 flex items-center gap-1.5 px-3 py-1.5 rounded-full
                 bg-err text-err-fg text-xs font-medium shadow-2
                 animate-in fade-in slide-in-from-bottom-2 duration-base"
    >
      <span className="w-1.5 h-1.5 rounded-full bg-err-fg opacity-80" aria-hidden="true" />
      Offline
    </div>
  );
}

 * tests/pwa/sw-toast.test.ts
 *
 * Unit tests for the SwRegister component's "new version available" toast logic.

 * We cannot mount the real SwRegister component in Vitest because it depends on:

// Extract the pure "new version available" handler logic
// ---------------------------------------------------------------------------
// Rather than importing SwRegister (which pulls in browser-only deps), we
// replicate the exact handler logic that lives inside the 'waiting' event
// listener. This keeps the test fast and dependency-free.

const TOAST_DELAY_MS = 3000;

type ToastAction = {
  label: string;
  onClick: () => void;
};

function createWaitingHandler(
  toastFn: (msg: string, opts: { id: string; duration: number; action: ToastAction }) => void,
  wbMessageSkipWaiting: () => void,
  reloadFn: () => void,
  delayMs: number = TOAST_DELAY_MS,
) {
  return () => {
    const timer = setTimeout(() => {
      toastFn('A new version of WhatToEat is ready.', {
        id: 'sw-update',
        duration: Infinity,
        action: {
          label: 'Refresh',
          onClick: () => {
            wbMessageSkipWaiting();
            reloadFn();
          },
        },
      });
    }, delayMs);
    return () => clearTimeout(timer);
  };
}

// ---------------------------------------------------------------------------
// Tests
// ---------------------------------------------------------------------------

describe('SW update toast — new version waiting handler', () => {
  beforeEach(() => {
    vi.useFakeTimers();
  });

  afterEach(() => {
    vi.useRealTimers();
  });

  it('does NOT show a toast immediately', () => {
    const toastFn = vi.fn();
    const handler = createWaitingHandler(toastFn, vi.fn(), vi.fn(), TOAST_DELAY_MS);
    handler();

    // Advance time by less than the delay
    vi.advanceTimersByTime(TOAST_DELAY_MS - 1);
    expect(toastFn).not.toHaveBeenCalled();
  });

  it('shows the toast after TOAST_DELAY_MS', () => {
    const toastFn = vi.fn();
    const handler = createWaitingHandler(toastFn, vi.fn(), vi.fn(), TOAST_DELAY_MS);
    handler();

    vi.advanceTimersByTime(TOAST_DELAY_MS);
    expect(toastFn).toHaveBeenCalledOnce();

    const handler = createWaitingHandler(toastFn, vi.fn(), vi.fn(), TOAST_DELAY_MS);
    handler();

    vi.advanceTimersByTime(TOAST_DELAY_MS);
    const opts = toastFn.mock.calls[0]![1] as { action: ToastAction };
    expect(opts.action.label).toBe('Refresh');
  });

    const toastFn = vi.fn();

    handler();

    vi.advanceTimersByTime(TOAST_DELAY_MS);

    expect(reload).toHaveBeenCalledOnce();
  });

  it('cleanup function cancels the timer', () => {

    const cleanup = handler();

 * scripts/assert-pwa-build.ts
 *

 *

 *   1. public/manifest.webmanifest is valid JSON with required PWA fields.
 *   2. public/sw.js exists and is non-empty.
 *   3. public/sw.js contains Workbox injection markers.

import fs from 'node:fs';
import path from 'node:path';

let failures = 0;

function assert(condition: boolean, message: string): void {
  if (condition) {
    console.log(`  PASS  ${message}`);
  } else {
    console.error(`  FAIL  ${message}`);
    failures++;
  }
}

function readJson(filePath: string): Record<string, unknown> | null {
  try {
    return JSON.parse(fs.readFileSync(filePath, 'utf8')) as Record<string, unknown>;
  } catch {
    return null;
  }
}

// ---------------------------------------------------------------------------
// 1. Manifest assertions
// ---------------------------------------------------------------------------

console.log('\nChecking manifest.webmanifest...');

const manifestPath = path.join(ROOT, 'public', 'manifest.webmanifest');
assert(fs.existsSync(manifestPath), 'manifest.webmanifest exists');

const manifest = readJson(manifestPath);
assert(manifest !== null, 'manifest.webmanifest is valid JSON');

if (manifest) {
  assert(typeof manifest.name === 'string' && manifest.name.length > 0, 'manifest.name is set');
  assert(typeof manifest.short_name === 'string', 'manifest.short_name is set');
  assert(typeof manifest.start_url === 'string', 'manifest.start_url is set');
  assert(
    manifest.display === 'standalone' || manifest.display === 'fullscreen',
    'manifest.display is standalone or fullscreen',
  );
  assert(Array.isArray(manifest.icons), 'manifest.icons is an array');

  const icons = manifest.icons as Array<{ src: string; sizes: string; purpose?: string }>;
  assert(
    icons.some((i) => i.sizes === '192x192'),
    'manifest has a 192x192 icon',
  );
  assert(
    icons.some((i) => i.sizes === '512x512'),
    'manifest has a 512x512 icon',
  );
  assert(
    icons.some((i) => i.purpose?.includes('maskable')),
    'manifest has at least one maskable icon',
  );
  assert(typeof manifest.theme_color === 'string', 'manifest.theme_color is set');
}

// ---------------------------------------------------------------------------
// 2. Service worker assertions
// ---------------------------------------------------------------------------

console.log('\nChecking public/sw.js...');

const swPath = path.join(ROOT, 'public', 'sw.js');
assert(fs.existsSync(swPath), 'public/sw.js exists (Workbox plugin ran)');

if (fs.existsSync(swPath)) {
  const swContent = fs.readFileSync(swPath, 'utf8');
  assert(swContent.length > 1024, 'public/sw.js is non-trivially sized (> 1 KB)');
  assert(
    swContent.includes('workbox') || swContent.includes('__WB_MANIFEST'),
    'public/sw.js contains Workbox markers',
  );
  assert(
    !swContent.includes('self.__WB_MANIFEST'),
    'self.__WB_MANIFEST placeholder has been replaced by injectManifest',
  );
}

// ---------------------------------------------------------------------------
// 3. Icon file assertions
// ---------------------------------------------------------------------------

console.log('\nChecking PWA icon files...');

const expectedIcons = [
  'icon-192.png',
  'icon-512.png',
  'icon-maskable-192.png',
  'icon-maskable-512.png',

  }
}

// ---------------------------------------------------------------------------
// Summary
// ---------------------------------------------------------------------------

console.log('');
if (failures === 0) {
  console.log('All PWA build assertions passed.');
  process.exit(0);
} else {
  console.error(`${failures} assertion(s) failed. Fix the issues above before merging.`);

import { test, expect } from '@playwright/test';

test.describe('PWA manifest and service worker', () => {
  test('manifest link is present in <head>', async ({ page }) => {
    await page.goto('/');
    const manifestLink = page.locator('link[rel="manifest"]');
    await expect(manifestLink).toHaveAttribute('href', '/manifest.webmanifest');
  });

  test('manifest endpoint returns 200 with correct content-type', async ({ request }) => {
    const response = await request.get('/manifest.webmanifest');
    expect(response.status()).toBe(200);
    const contentType = response.headers()['content-type'] ?? '';
    expect(contentType).toContain('application/manifest+json');
  });

  test('manifest JSON has required fields', async ({ request }) => {
    const response = await request.get('/manifest.webmanifest');
    const manifest = await response.json() as Record<string, unknown>;
    expect(manifest.name).toBeTruthy();
    expect(manifest.short_name).toBeTruthy();
    expect(manifest.start_url).toBeTruthy();
    expect(manifest.display).toBe('standalone');
    const icons = manifest.icons as Array<{ sizes: string; purpose?: string }>;
    expect(icons.some((i) => i.sizes === '512x512')).toBe(true);
    expect(icons.some((i) => i.purpose?.includes('maskable'))).toBe(true);
  });

  test('sw.js is served with correct content-type and Service-Worker-Allowed header', async ({ request }) => {
    const response = await request.get('/sw.js');
    expect(response.status()).toBe(200);
    const contentType = response.headers()['content-type'] ?? '';
    expect(contentType).toContain('application/javascript');
    expect(response.headers()['service-worker-allowed']).toBe('/');
  });

  test('service worker registers and activates', async ({ page }) => {
    await page.goto('/');

    // Wait for the SW to register and activate (up to 5 seconds).
    const swState = await page.evaluate(async () => {
      if (!('serviceWorker' in navigator)) return 'unsupported';
      const reg = await navigator.serviceWorker.ready;
      return reg.active?.state ?? 'no-active';
    });

    expect(swState).toBe('activated');
  });

  test('offline page renders when navigated to directly', async ({ page }) => {
    await page.goto('/offline');
    await expect(page.getByRole('heading', { name: "You're offline" })).toBeVisible();
    await expect(page.getByRole('button', { name: 'Try again' })).toBeVisible();
    await expect(page.getByRole('link', { name: 'pantry' })).toBeVisible();
  });

  test('apple-mobile-web-app-capable meta is set', async ({ page }) => {
    await page.goto('/');
    const meta = page.locator('meta[name="apple-mobile-web-app-capable"]');
    await expect(meta).toHaveAttribute('content', 'yes');
  });

  test('theme-color meta is set to brand amber', async ({ page }) => {
    await page.goto('/');
    const meta = page.locator('meta[name="theme-color"]');
    // Next.js 15 emits multiple theme-color meta tags for light/dark variants.
    // We assert at least one of them contains our amber color.
    const count = await meta.count();
    expect(count).toBeGreaterThanOrEqual(1);
    const firstContent = await meta.first().getAttribute('content');
    expect(firstContent?.toLowerCase()).toMatch(/#d97706|#fbbf24/i);
  });
});
